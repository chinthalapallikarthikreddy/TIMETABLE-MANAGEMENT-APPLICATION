# Requirements Management (REQM)

## Purpose
The objective of Requirements Management is to ensure that project requirements are clearly defined, agreed upon, and controlled throughout the development lifecycle.

## Application in Project
- Functional requirements were captured as GitHub user stories.
- Each user story includes acceptance criteria.
- Changes to requirements were handled through backlog refinement.

## Evidence
- GitHub Issues (User Stories)
- Sprint Backlog in GitHub Projects
- Acceptance criteria embedded in issue descriptions
